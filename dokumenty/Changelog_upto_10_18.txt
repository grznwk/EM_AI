Changelog
10.18 (2026-02-19)

Dedicated exports
Enables dedicated export formats for individual accounts/users with custom-tailored data structures to meet specific business needs.
Inclusion of "type specification" in the attachment type across the application
Attachment type management has been standardized throughout the application to a single-level structure combining: type (ATX) + type specification (MDX).
Attachment type descriptions in the Excel "Info" header section
Descriptive labels for attachment types have been added to the first row of the Excel file header ("Info" header). This enhancement facilitates identification for users unfamiliar with ATX/MDX coding conventions.
Change in the display of multiple items in Excel
The minimum number of exported columns (or column sections) for multiple elements has been reduced from 2 to 1. Previously, at least two columns were always generated to indicate the multiplicity of a given element.
Bulk "Mark as read"
This feature helps you to mark multiple selected messages as read, to focus on the conversations that matter most.
10.17 (2026-02-10)

ATX/MDX reference level
Attachment referencing has been extended to support the ATX_MDX level for attachment types (ATX) that are divided into specification types (MDX). This change affects multiple areas of the application, including export/import, validation, group modification, filtering, and attachment display (application UI and product cards).
Attachment group definition change (PDC)
The definition of attachment groups in custom company completeness masks has been updated. Arbitrarily defined administrator-created attachment groups have been replaced with a user-defined selective choice of ATX_MDX values.
10.16 (2026-01-29)

Dependency of "Material category" on "Material type" in packaging materials
Implemented a hierarchical relationship between "Material type" and "Material category" for packaging materials. Selecting a material type now filters the available material categories to show only relevant options. Note: the "Other materials" type remains without an associated category.
Automatic setting of the "Type specification" field in attachments when empty
Attachment type specification (MDX) field is conditionally required based on the ATX field. To ensure data integrity, any missing MDX values will be defaulted to MDX999.
10.15 (2026-01-23)

Refactoring of ETIM feature suggestion rules
Optimization and cleanup of the logic that governs how specific ETIM features are suggested. One of the unused and burdensome rules has been removed: "Matching based on previous classifications of other products".
FAQ section
Introduced a FAQ section to the documentation to provide quick answers to common technical queries and system usage questions.
10.14 (2026-01-20)

Adding packaging materials to packings
The xChange 2.0 integration has been finalized and now includes the ability to define packaging materials.
Removing "Packing parts" from packings
The optional "Packing parts" field in the packing definition has been removed. In xChange, this field corresponds to: "NumberOfPackagingParts". The application ignores cases where a single product is delivered in several packages; these must be treated as a whole (dimensions, weight, materials, etc.). Dealing with multiple elements at every level: multiple packing parts per product, multiple packings per product, and multiple materials per packing part — creates significant complexity. We have limited the structure to two levels: packings and materials.
Removing "ETIM classification language"
The option to select the ETIM language independently of the interface language has been discontinued, as ETIM translations were previously unavailable in some languages. Currently, the language selection applies to the entire application (interface and ETIM).
10.13 (2025-12-30)

Adaptation of the application to xChange 2.0
Enabling import/export compatible with xChange 2.0, including adding new fields and adapting field types, formats, and names to the new standard (list of changes).
10.12 (2025-12-10)

AI product description generation
A new paid option has been added that enables AI-based content generation and translation for multilingual fields.
10.11 (2025-11-21)

Merging of languages for the interface, product descriptions and ETIM descriptions
With the introduction of AI-based translations, the previously separate management of languages for the interface, product descriptions and ETIM descriptions — resulting from the unavailability of certain content in selected languages (e.g., ETIM descriptions) — has been merged into a single, consistent language configuration. English remains the reference language as it is available in every account. Other languages may contain translations generated automatically by AI, which may not always be entirely correct, especially in the case of ETIM terminology. If in doubt about the meaning of a particular term, use the English version as the authoritative source.
10.10 (2025-11-14)

File syntax validation during xChange/BMEcat file import
Before import, xChange/BMEcat files are checked for syntactic correctness, and only correctly structured files are imported. This prevents partial imports.
10.9 (2025-11-07)

Validation of xChange files
The application has been enhanced with a function for validating xChange files. It checks the correctness of the JSON structure (format correctness and validity), as well as compliance with the schema and the ETIM xChange Guidelines for product data delivery.
10.8 (2025-09-30)

File validation now available for standard and comfort accounts
Validating a file before import (available on the "Files / Validation" page) helps ensure that the data structure and format are correct. This makes the import process more predictable and reduces the risk of unexpected errors or failed uploads. It’s a simple step that can save time and prevent issues.
"ETIM Override" feature has been made public for standard and comfort accounts
This function allows you to override the official names of selected ETIM elements to adapt them to the specifics of your own products in the application or on product cards. The configuration is available in the "Settings / ETIM" page.
LCA functionality is now accessible for standard and comfort accounts
The LCA function allows you to indicate reference indicators that help assess the environmental impact of a product throughout its entire life cycle (if not visible, you can enable it in "Settings / Account / Product editing tabs").
New procedure for listing files from the disk space (for standard and comfort accounts)
Depending on the selected mode, the procedure allows you to list: all files, files used in ATX attachment paths, or files not used in attachment paths. Note: files not used in attachment paths may still be used by external applications — caution is advised before deleting them.
10.7 (2025-09-17)

User defined validations for attributes
An option for dedicated validation of user-defined attributes has been introduced. Only attribute values that meet the requirements specified for a given account are accepted. This validation is performed on request and after agreement with the user. Attributes subject to validation become system attributes (they cannot be deleted).
10.6 (2025-09-03)

ETIM class explanations
Explanations for individual ETIM classes have been added to the ETIM classification tab when editing a single product. These explanations clarify which products fall under a given class and assist in selecting the appropriate class for products.
10.5 (2025-08-25)

Recipient completeness mask
For accounts with the recipient option enabled, it is now possible to enable dedicated completeness masks for recipients. From the recipient’s perspective, such a mask appears as a "custom company mask". Each recipient cannot see the dedicated masks of other recipients or the main account’s dedicated mask. A recipient can only use the default mask and their own dedicated mask. Each dedicated recipient mask is initially based on the settings of the main account’s dedicated mask.
10.4 (2025-08-04)

External links
A new feature allows managers to add a personalized menu visible on the dashboard. The items in this menu serve as links to specified pages containing relevant additional information or downloadable content.
Update to the custom company completeness mask within the ETIM section
The option to selectively choose required features within individual classes — previously unused in practice — has been discontinued in favor of mandatory or optional description at the entire ETIM classification level. When ETIM is set as mandatory, assigning a class to a product accounts for 10% of the ETIM description completeness. The remaining 90% comes from features, all of which are required.
Consolidation of ETIM settings
All ETIM-related options, including the ETIM override, have been consolidated into a single section.
Possibility to import/export "As supplier" for supplier users
The "As supplier" export option has been enabled for supplier users. It allows "Product ID by supplier" to be used as primary identifiers in both exported and imported files.

10.1 (2025-06-24)

It is also possible to modify an attachment with a given URI on multiple products: attachments can be added to products, their values can be replaced, or they can be deleted.

Sorting text fields according to language in product views

10.0 (2025-05-09)

ETIM version 10
The latest version (version 10) of the ETIM classification has been made available in the application. Reference images and class synonyms have been updated. ETIM v10 is now the default version for xChange/BMEcat files and the working version for Excel files.
Automatic products migration to ETIM version 10
The technical description of the products has been migrated to ETIM version 10. The migration assistant ("ETIM v10 migration" view) indicates products that have been changed in some way when adapting them to the new ETIM classification. The migration assistant displays deleted or modified features next to their successors, facilitating comparison and acceptance of the changes made. Migrating products does not change the settings of xChange/BMEcat export links. Existing xChange/BMEcat export links using ETIM v9 will continue to provide technical information in ETIM v9. To make technical information available in the new ETIM version, a new xChange/BMEcat export link must be generated and made available. ETIM v10 is always used to export/import Excel files.

These changes aim to streamline communication, improve tracking, and provide clearer status information for all involved users.
9.39 (2025-01-08)

By default, the dashboard is set as the homepage. This can be changed in "Settings / Account" page.
Priorities in submissions to administrators

9.35 (2024-11-14)

Renaming "user-defined mask" to "custom company mask"
The change was made due to a misleading name (user-defined may suggest that the mask is defined individually for each user, whereas it is a mask for the whole company). This implies a change in the name of the Excel tab: from "User-defined completeness mask" to "Custom company mask" and a change in the name of the Excel field: from "User-defined mask [%]" to "Custom company mask [%]".
Option to enable notifications of a new version for individual users
The ability to enable notifications for application changes has been added in the user settings. This can now be configured individually for each user.
Marking stagnant submissions to suppliers (tasks with no progress)
Open task submissions in which no progress is observed are marked in red ("low" priority is considered stagnant if the last entry is older than 7 days; "medium" - if the last entry is older than 3 days and "high" priority - if the last entry is older than 1 day)
Indication of whether an entry (submission to suppliers) has been read
To facilitate the follow-up of submissions to suppliers, an indication of whether individual entries have been read has been introduced.
XChange 1.1
The application now supports export and import in the xChange format version 1.1.

The new look of selecting labels allows them to be more easily identified and facilitates quick choice of the correct subset of labels (especially if there are many of them), which was a weak point in the previous interface.
Export sub-menu
Due to the increasing number of export options, they have been separated by the type of exported files and placed in sub-menus ("BMEcat", "Excel", "System product cards", "Local product cards").
First column of product description locked when scrolling the columns to the right (all product views)
Locking the first column in the product view makes it easier to identify products when scrolling the columns to the right. Instead of the main product identifier, you can place another identifier (e.g. GTIN) in the first column - to do this, use the column reordering option in the hamburger menu.
Possibility to add an additional column with a selected product identifier (Excel export)
This option allows to include a column containing an additional product identifier (next to the main product identifier) in the exported Excel file. It is intended to facilitate the identification of products in communication between companies that usually refer to different product identifiers. This additional column is omitted during import, which is always based on the main product ID.
Clone users by regional managers
With this option, regional managers can clone existing users (a useful option when creating multiple users with similar authorizations).
